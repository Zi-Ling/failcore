# FailCore

**FailCore 是 AI agent 的执行时安全运行时。**

它不分析提示。  
它不调试推理。  
它不重写工具调用。

FailCore 只回答一个问题：

> **这个操作现在应该被允许执行吗？**

---

## 为什么需要 FailCore

现代 AI agent 经常失败**不是在推理中**，而是**在执行时**。

即使 agent 产生了一个看似正确的计划，它仍然可能：
- 产生错误的文件系统路径
- 向内部 IP 发送网络请求
- 通过流式输出泄露密钥
- 进入无限工具循环并消耗成本

这些失败发生在**规划之后**，在操作执行的确切时刻。

FailCore 的存在是为了填补这个执行时空白。

---

## FailCore 做什么

FailCore 位于**agent 意图**和**副作用**之间。

它对操作强制执行确定性的、策略驱动的边界，例如：

- **文件系统访问**  
  防止破坏性或超出范围的读取/写入。
- **网络调用**  
  阻止 SSRF、内部 IP 访问和数据泄露。
- **工具执行**  
  在允许工具运行之前验证参数。
- **成本和资源使用**  
  停止失控循环和不受控制的支出。
- **流式输出**  
  在实时 token 流中检测违规。

所有决策都使用**可追溯的证据**记录，用于审计和重放。

---

## FailCore 不是什么

FailCore **不是**：
- 提示工程框架
- agent 规划器或推理调试器
- 沙箱替代品（Docker / VM）
- 模型行为的策略语言

FailCore **不**告诉 agent *思考什么*。  
它决定**是否允许操作执行**。

---

## 何时应该使用 FailCore

FailCore 设计用于：

- 可以执行真实世界副作用的 AI agent
- 长时间运行或自主工作流
- 触及文件系统、网络或外部 API 的工具
- 失败必须可解释和可审计的环境

如果 agent 可能造成损害，FailCore 应该在执行路径中。
