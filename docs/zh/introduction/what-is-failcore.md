# 什么是 FailCore

FailCore 是 AI agent 的**执行时安全运行时**。

它不分析提示。  
它不调试推理。  
它不重写工具调用。

FailCore 只回答一个问题：

> **这个操作现在应该被允许执行吗？**

---

## 核心定位

FailCore 位于 **agent 意图** 和 **副作用** 之间。

当 AI agent 生成工具调用时，FailCore 在执行之前进行验证：

- ✅ **执行时强制执行** — 在操作运行前阻止不安全操作
- ✅ **结构化追踪** — 每个操作都作为证据记录
- ✅ **清晰的结果** — `BLOCKED` vs `FAIL` 明确区分
- ✅ **取证报告** — 离线检查事件
- ✅ **代理优先设计** — 观察真实流量，而非演示

---

## 工作原理

FailCore 作为 **运行时安全层** 运行在：

- LLM 框架 / SDK
- 和真实世界（文件系统、网络、进程）

之间。

它**不是** agent 框架。  
它是**护栏、记录器和黑匣子**。

---

## 解决的问题

现代 AI agent 经常失败**不是在推理中**，而是**在执行时**。

即使 agent 产生了一个看似正确的计划，它仍然可能：

- ❌ 产生错误的文件系统路径
- ❌ 向内部 IP 发送网络请求
- ❌ 通过流式输出泄露密钥
- ❌ 进入无限工具循环并消耗成本

这些失败发生在**规划之后**，在操作执行的确切时刻。

FailCore 的存在是为了填补这个执行时空白。

---

## 主要功能

FailCore 对操作强制执行确定性的、策略驱动的边界：

### 文件系统访问
防止破坏性或超出范围的读取/写入。

### 网络调用
阻止 SSRF、内部 IP 访问和数据泄露。

### 工具执行
在允许工具运行之前验证参数。

### 成本和资源使用
停止失控循环和不受控制的支出。

### 流式输出
在实时 token 流中检测违规。

所有决策都使用**可追溯的证据**记录，用于审计和重放。

---

## FailCore 不是什么

FailCore **不是**：

- 提示工程框架
- agent 规划器或推理调试器
- 沙箱替代品（Docker / VM）
- 模型行为的策略语言

FailCore **不**告诉 agent *思考什么*。  
它决定**是否允许操作执行**。

---

## 项目状态

FailCore 正在积极开发中（0.1.x 预发布版本）。

API、CLI 命令和报告格式可能会发生变化。

当前专注于：
- 确定性执行
- 快速失败行为
- 最小、可执行的保证
