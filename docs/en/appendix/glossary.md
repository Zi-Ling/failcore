# Glossary

This document defines terms used in FailCore.

---

## A

### Audit

The process of examining and analyzing execution records.

### Allowlist

A list of explicitly allowed operations or resources.

---

## B

### Boundary

A declarative specification that defines the allowed scope of side effects.

### Blocked

The state where policy check fails and execution is blocked.

---

## C

### Cost Guardian

A component that monitors and controls execution costs.

### Context

The execution environment containing policy, trace, sandbox, and other configurations.

---

## D

### Deny

Policy check fails, execution not allowed.

---

## E

### Execution Boundary

A boundary that defines the allowed scope of operations.

### Executor

The component responsible for executing tool calls.

---

## F

### Fail

The state where tool execution errors occur.

### FailCore

An execution-time safety runtime for AI agents.

---

## G

### Guard

A decorator or function that protects tool calls.

---

## M

### MCP (Model Context Protocol)

A protocol for LLMs to communicate with external tools.

### Middleware

Components that execute before and after tool calls.

---

## P

### Policy

Configuration that defines security rules.

### PolicyResult

The result of a policy check.

### Preset

Pre-configured policy templates.

---

## R

### Replay

The process of re-executing or analyzing using historical traces.

### Run

A complete execution session.

### Run Context

A context object containing execution environment information.

---

## S

### Sandbox

A directory that restricts filesystem operations.

### Side Effect

Changes to the external world when tools execute.

### SSRF (Server-Side Request Forgery)

Server-side request forgery attacks.

### Strict

Mode where policy violations raise exceptions.

---

## T

### Trace

A complete record of the execution process.

### Tool

An executable function or operation.

---

## V

### Validator

Components that perform policy checks.

### Violation

Behavior that violates policy rules.

---

## Summary

These terms are frequently used in FailCore documentation. If you encounter unfamiliar terms, please refer to this table.
